Linux Command: ifconfig

Configure and display network interface information

Home top kill ifconfig ping ssh scp curl diff head tail

Command Overview

The ifconfig command is used to configure network interfaces and display their current configuration. While it's being deprecated in favor of the ip command, it's still widely used and available on most systems.

Basic Syntax

ifconfig                    # Display all interfaces
ifconfig interface          # Display specific interface
ifconfig interface up/down  # Enable/disable interface
ifconfig interface options  # Configure interface

Common Options

  • up/down - Enable or disable interface
  • netmask - Set network mask
  • broadcast - Set broadcast address
  • mtu - Set Maximum Transmission Unit
  • -a - Show all interfaces, even if down

Common Examples

# Show all network interfaces
ifconfig

# Configure IP address and netmask
ifconfig eth0 192.168.1.100 netmask 255.255.255.0

# Enable interface
ifconfig eth0 up

# Disable interface
ifconfig eth0 down

# Set MTU size
ifconfig eth0 mtu 1500

# Show only active interfaces
ifconfig -a

Interface Flags

  • UP - Interface is active
  • BROADCAST - Supports broadcasting
  • RUNNING - Interface ready to accept data
  • MULTICAST - Supports multicasting
  • LOOPBACK - Loopback interface
  • POINTOPOINT - Point-to-point link
  • PROMISC - Promiscuous mode
  • NOARP - No ARP protocol

Advanced Usage

# Set interface MTU
ifconfig eth0 mtu 9000

# Enable promiscuous mode
ifconfig eth0 promisc

# Disable promiscuous mode
ifconfig eth0 -promisc

# Set interface metric
ifconfig eth0 metric 100

# Set transmit queue length
ifconfig eth0 txqueuelen 2000

# Enable multicast
ifconfig eth0 multicast

# Set hardware class
ifconfig eth0 hw ether 00:11:22:33:44:55

Network Configuration Examples

# Configure IPv4 with netmask and broadcast
ifconfig eth0 192.168.1.10 netmask 255.255.255.0 broadcast 192.168.1.255

# Add secondary IP address
ifconfig eth0:1 192.168.2.10 up

# Configure IPv6 address
ifconfig eth0 inet6 add 2001:db8::1/64

# Remove IPv6 address
ifconfig eth0 inet6 del 2001:db8::1/64

# Set interface up with specific parameters
ifconfig eth0 up mtu 1500 txqueuelen 1000

Troubleshooting

  • No carrier:
    • Check physical connection
    • Verify cable integrity
    • Check switch/router port
  • TX errors:
    • Check for collisions
    • Verify duplex settings
    • Monitor network congestion
  • RX errors:
    • Check for packet corruption
    • Verify buffer sizes
    • Monitor for overruns

Performance Tuning

# Optimize for jumbo frames
ifconfig eth0 mtu 9000

# Adjust ring buffer
ethtool -G eth0 rx 4096 tx 4096

# Set interface speed and duplex
ethtool -s eth0 speed 1000 duplex full

# Monitor interface statistics
watch -n 1 "ifconfig eth0 | grep bytes"

Scripting Examples

# Monitor interface bandwidth
while true; do
    RX_BYTES=$(ifconfig eth0 | grep "RX bytes" | awk '{print $2}' | cut -d: -f2)
    sleep 1
    RX_BYTES_NEW=$(ifconfig eth0 | grep "RX bytes" | awk '{print $2}' | cut -d: -f2)
    echo "Bandwidth: $(((RX_BYTES_NEW-RX_BYTES)/1024)) KB/s"
done

# Check interface status
check_interface() {
    if ifconfig $1 | grep -q "UP"; then
        echo "Interface $1 is up"
    else
        echo "Interface $1 is down"
    fi
}

# Reset interface
reset_interface() {
    ifconfig $1 down
    sleep 2
    ifconfig $1 up
}

Modern Alternatives (ip command)

# Show interfaces
ip link show

# Set interface up/down
ip link set eth0 up/down

# Set IP address
ip addr add 192.168.1.10/24 dev eth0

# Show interface statistics
ip -s link show eth0

# Add route
ip route add default via 192.168.1.1

# Show neighbors
ip neigh show

Modern Alternative: ip Command

# Show interfaces (ip equivalent)
ip addr show

# Configure IP (ip equivalent)
ip addr add 192.168.1.100/24 dev eth0

# Enable/disable (ip equivalent)
ip link set eth0 up/down

# Show statistics (ip equivalent)
ip -s link show eth0

Pro Tips

  • Use ip command for modern systems
  • Check interface status before configuration
  • Backup current configuration before changes
  • Use ifconfig -a to see all interfaces
  • Remember to configure DNS separately

⚠️ Important Notes

  • Changes made with ifconfig are temporary
  • For permanent changes, edit network configuration files
  • Some systems may require root privileges
  • The command is deprecated on modern systems
