Linux Command: head

Display the beginning of files and preview file content

Command Overview

The head command displays the beginning of a text file or piped data. It's useful for previewing large files or data streams by showing the first few lines by default.

Basic Syntax

head file              # Show first 10 lines
head -n N file         # Show first N lines
head -c N file         # Show first N bytes

Common Options

  • -n, --lines - Output the first N lines
  • -c, --bytes - Output the first N bytes
  • -q, --quiet - Suppress file headers
  • -v, --verbose - Show file headers

Common Examples

# Show first 10 lines
head file.txt

# Show first 50 lines
head -n 50 file.txt

# Show first 100 bytes
head -c 100 file.txt

# Show first 20 lines without headers
head -n 20 -q file1.txt file2.txt

# Show first 5 lines with file headers
head -n 5 -v file1.txt file2.txt

Basic Usage

Option Description Example
-n N Display first N lines head -n 20 file.txt
-c N Display first N bytes head -c 100 file.txt
-q Suppress file headers head -q file1.txt file2.txt
-v Show file headers head -v file.txt

Advanced Usage

# Show first 100 lines and suppress headers
head -n 100 -q file1.txt file2.txt

# Display first 1KB of data
head -c 1024 file.bin

# Show first 10 lines of multiple files
head file1.txt file2.txt

# Combine head and grep
head -n 50 log.txt | grep 'ERROR'

Scripting Examples

# Process first N lines
process_head() {
    head -n "$1" "$2" | while read -r line; do
        # Process each line
        echo "[$(date)]: $line"
    done
}

# Preview and notify
preview_notify() {
    head -n 10 "$1" | while read -r line; do
        notify-send "File Preview" "$line"
    done
}

# Batch head processing
batch_head() {
    for file in "$@"; do
        echo "=== First 10 lines of $file ==="
        head -n 10 "$file"
        echo
    done
}

Performance Considerations

  • CPU Usage:
    • Low CPU usage
    • Quick file access
  • File System Impact:
    • Minimal disk I/O
    • Works well with network filesystems

Troubleshooting

  • Common Issues:
    • Incorrect line count
    • File permission problems
  • Solutions:
    • Verify line count
    • Check file permissions

Best Practices

  • Use -n to specify line count
  • Combine with grep for filtering
  • Suppress headers when processing multiple files
  • Monitor resource usage when dealing with large files
