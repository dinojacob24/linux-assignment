Linux Command: ping

Command Overview

The ping command sends ICMP ECHO_REQUEST packets to network hosts. It's used to test network connectivity, measure response times, and diagnose network issues.

Basic Syntax

ping host               # Continuous ping
ping -c count host      # Ping specific number of times
ping -i interval host   # Set interval between pings
ping -s size host      # Set packet size

Common Options

  • -c count - Stop after sending count packets
  • -i interval - Wait interval seconds between pings
  • -s size - Specify the packet size
  • -q - Quiet output, only show summary
  • -w deadline - Stop after deadline seconds

Common Examples

# Ping Google's DNS server 4 times
ping -c 4 8.8.8.8

# Ping with 2-second intervals
ping -i 2 example.com

# Ping with larger packet size
ping -s 1000 192.168.1.1

# Ping with deadline
ping -w 10 example.com

# Quiet ping with summary
ping -q -c 5 example.com

Understanding Output

PING example.com (93.184.216.34) 56(84) bytes of data.
64 bytes from 93.184.216.34: icmp_seq=1 ttl=56 time=11.6 ms
64 bytes from 93.184.216.34: icmp_seq=2 ttl=56 time=12.1 ms

--- example.com ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 11.629/11.866/12.104/0.237 ms
  • icmp_seq - Sequence number of the packet
  • ttl - Time To Live value
  • time - Round-trip time in milliseconds
  • packet loss - Percentage of packets lost
  • rtt - Round-trip time statistics

ICMP Packet Fields

Field Description
bytes Size of ICMP packet
ttl Time To Live value
time Round-trip time in ms
seq Sequence number
icmp_seq ICMP sequence number

Advanced Usage

# Specify packet size
ping -s 1500 example.com

# Set TTL value
ping -t 64 example.com

# Set timeout
ping -W 2 example.com

# Flood ping (requires root)
ping -f example.com

# Custom interval
ping -i 0.2 example.com

# Specify interface
ping -I eth0 example.com

# Record route
ping -R example.com

# Timestamp request
ping -T tsonly example.com

Network Diagnostics

# Check network connectivity
ping -c 4 8.8.8.8

# Measure packet loss
ping -c 100 -i 0.2 server.com | grep "packet loss"

# Monitor latency
ping -D example.com

# Check MTU issues
ping -M do -s 1472 example.com

# Audible ping
ping -a example.com

# Quiet output
ping -q -c 5 example.com

Scripting Examples

# Monitor host availability
while true; do
    if ping -c 1 -W 1 host.com >/dev/null; then
        echo "$(date): Host is up"
    else
        echo "$(date): Host is down"
    fi
    sleep 60
done

# Check multiple hosts
check_hosts() {
    for host in "$@"; do
        if ping -c 1 -W 1 "$host" >/dev/null; then
            echo "$host is reachable"
        else
            echo "$host is unreachable"
        fi
    done
}

# Monitor latency with threshold
monitor_latency() {
    threshold=$1
    host=$2
    ping -c 1 $host | awk -v threshold=$threshold '
        /time=/ {
            split($7, a, "=")
            if (a[2] > threshold)
                print "High latency: " a[2] "ms"
        }'
}

Performance Analysis

# Calculate average RTT
ping -c 100 example.com | tail -1 | awk '{print $4}' | cut -d '/' -f 2

# Monitor jitter
ping example.com | awk '
    /time=/ {
        split($7, a, "=")
        if (last) {
            jitter = a[2] - last
            if (jitter < 0) jitter = -jitter
            print "Jitter: " jitter "ms"
        }
        last = a[2]
    }'

# Extended statistics
ping -c 10 -q example.com | awk '
    /min\/avg\/max/ {
        split($4, a, "/")
        print "Min: " a[1] "ms"
        print "Avg: " a[2] "ms"
        print "Max: " a[3] "ms"
    }'

Best Practices

  • Use appropriate packet sizes
  • Set reasonable timeouts
  • Consider rate limiting
  • Monitor both IPv4 and IPv6
  • Document baseline metrics
  • Use multiple test points
  • Analyze patterns over time
  • Keep security in mind

Pro Tips

  • Use -c to limit the number of pings
  • Higher response times indicate network congestion
  • Packet loss suggests network problems
  • Some hosts may block ICMP packets
  • Use ping6 for IPv6 addresses
