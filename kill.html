Linux Command: kill

Terminate or send signals to processes

Home top kill ifconfig ping ssh scp curl diff head tail

Command Overview

The kill command is used to send signals to processes. While its name implies termination, it can actually send various signals to processes for different purposes.

Basic Syntax

kill [options] PID         # Send signal to process by PID
kill -l                  # List all available signals
kill -SIGNAL PID         # Send specific signal by name
kill -s SIGNAL PID       # Send specific signal by name

Common Signals

  • SIGTERM (15) - Graceful termination (default)
  • SIGKILL (9) - Immediate termination
  • SIGHUP (1) - Hangup signal
  • SIGINT (2) - Interrupt signal (Ctrl+C)
  • SIGSTOP (19) - Stop process

Common Examples

# Gracefully terminate process 1234
kill 1234

# Force kill a process
kill -9 1234

# Kill all processes matching 'firefox'
killall firefox

# Send HUP signal to reload configuration
kill -HUP 1234

# Kill all processes of a specific user
killall -u username

# List all running processes of 'firefox'
pgrep firefox

Process Management Tools

# Find process ID by name
pidof process_name

# Find process details
pgrep -a process_name

# Kill all processes by name
pkill process_name

# Kill processes by pattern
pkill -f "pattern"

# Kill processes by user
pkill -u username

# Kill oldest process of a name
pkill -o process_name

# Kill newest process of a name
pkill -n process_name

Advanced Usage

# Kill process group
kill -TERM -pid

# Kill all processes except shell
kill -9 -1

# Send multiple signals
kill -STOP pid && sleep 5 && kill -CONT pid

# Kill processes older than time
find /proc -maxdepth 1 -user username -mmin +120 \
  -exec basename {} \; | xargs kill

# Kill by percentage CPU usage
ps aux | awk '$3 > 50 {print $2}' | xargs kill

# Kill zombie processes
ps aux | awk '$8 ~ /Z/ {print $2}' | xargs kill -9

Scripting Examples

# Monitor and kill high CPU processes
while true; do
    ps aux | awk '$3 > 90 {print $2}' | xargs kill
    sleep 60
done

# Graceful shutdown script
for pid in $(pgrep process_name); do
    kill -TERM $pid
    sleep 2
    if kill -0 $pid 2>/dev/null; then
        kill -KILL $pid
    fi
done

# Clean up defunct processes
for pid in $(ps -A -ostat,ppid | awk '/[zZ]/ {print $2}'); do
    kill -9 $pid
done

Troubleshooting

  • Process won't die:
    • Check if process is in uninterruptible sleep (D state)
    • Try SIGKILL (-9) as last resort
    • Check for zombie processes
    • Verify process ownership
  • Permission denied:
    • Verify process ownership
    • Check if sudo is required
    • Verify user permissions
  • No such process:
    • Verify PID is correct
    • Check if process has already terminated
    • Use pgrep to find correct PID

Best Practices

  • Always try SIGTERM first before SIGKILL
  • Verify process ID before killing
  • Use pkill/killall with caution
  • Check child processes
  • Document signal handlers in applications
  • Implement proper shutdown procedures
  • Monitor process states
  • Use appropriate signal for the task

Pro Tips

  • Always try SIGTERM (default) before using SIGKILL
  • Use killall to kill processes by name
  • Use pkill for more flexible process matching
  • Check process status with ps before killing
  • Use pgrep to find process IDs

⚠️ Important Warnings

  • SIGKILL (-9) should be used as a last resort
  • Killing system processes can destabilize your system
  • Always verify the PID before killing a process
  • Some processes may spawn new processes after being killed
