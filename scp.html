Linux Command: scp

Secure Copy - Transfer files securely between systems

Home top kill ifconfig ping ssh scp curl diff head tail

Command Overview

The scp (Secure Copy) command securely copies files between hosts on a network using SSH for data transfer. It provides encryption and authentication.

Basic Syntax

scp source target           # Basic copy
scp file user@host:path     # Copy to remote
scp user@host:file local    # Copy from remote
scp -r directory host:path  # Copy directory

Common Options

  • -r - Recursively copy directories
  • -P port - Specify port number
  • -i key - Identity file (private key)
  • -C - Enable compression
  • -p - Preserve file attributes

Common Examples

# Copy file to remote server
scp file.txt user@example.com:/path/to/destination

# Copy from remote to local
scp user@example.com:/path/to/file.txt local.txt

# Copy entire directory
scp -r directory/ user@example.com:/path/

# Copy with specific port
scp -P 2222 file.txt user@example.com:~

# Copy with compression
scp -C large_file.zip user@example.com:~

# Copy multiple files
scp file1.txt file2.txt user@example.com:~/files/

Transfer Options

Option Description Example
-r Recursive copy scp -r dir/ host:path/
-p Preserve modification times scp -p file.txt host:file.txt
-C Enable compression scp -C large.zip host:
-l limit Limit bandwidth (Kbit/s) scp -l 1000 file.txt host:
-P port Specify port number scp -P 2222 file.txt host:
-i identity_file Identity file for auth scp -i key.pem file.txt host:

Advanced Usage

# Copy with specific key and port
scp -i ~/.ssh/custom_key -P 2222 file.txt user@host:~/

# Copy with bandwidth limit
scp -l 1000 large_file.zip user@host:~/

# Preserve file attributes
scp -p important.txt user@host:~/

# Copy between remote hosts
scp user1@host1:/file user2@host2:/path/

# Copy multiple files
scp file1.txt file2.txt user@host:~/

# Copy excluding files
rsync --exclude='*.tmp' -e ssh src/ user@host:dest/

Performance Optimization

# Fast transfer configuration
scp -o 'Compression=yes' \
    -o 'CompressionLevel=9' \
    -o 'Ciphers=aes128-gcm@openssh.com' \
    large_file.txt host:

# Parallel transfer script
parallel_scp() {
    for file in "$@"; do
        scp "$file" user@host:~/uploads/ &
    done
    wait
}

# Resume interrupted transfer
rsync -P --rsh=ssh file.txt user@host:~/

# Monitor transfer speed
pv file.txt | ssh user@host 'cat > file.txt'

Scripting Examples

# Backup script with timestamp
backup_to_remote() {
    local date=$(date +%Y%m%d)
    tar czf - "$1" | \
    ssh user@host "cat > backup_${date}.tar.gz"
}

# Sync directories
sync_dirs() {
    local src="$1"
    local dest="$2"
    rsync -avz -e ssh "$src" "user@host:$dest"
}

# Multi-host transfer
for host in $(cat hosts.txt); do
    scp -o BatchMode=yes \
        file.txt "$host:~/data/"
done

Troubleshooting

  • Connection Issues:
    • Check SSH connectivity
    • Verify port accessibility
    • Check firewall rules
    • Test with verbose mode (-v)
  • Permission Problems:
    • Check file permissions
    • Verify directory permissions
    • Confirm user ownership
    • Check SSH key permissions
  • Performance Issues:
    • Enable compression
    • Use faster ciphers
    • Check network bandwidth
    • Monitor system resources

Best Practices

  • Use key-based authentication
  • Compress large transfers
  • Verify file integrity
  • Monitor transfer progress
  • Use bandwidth limits when needed
  • Implement error handling
  • Keep logs of transfers
  • Use rsync for large directories
  • Regular backup automation
  • Secure key management

Modern Alternatives

  • rsync: Better for synchronization 
    rsync -avz -e ssh source/ user@host:dest/
  • sftp: Interactive file transfer 
    sftp user@host
  • rclone: Cloud storage support 
    rclone copy source:path dest:path
  • git: Version-controlled transfers 
    git clone ssh://user@host/path/repo.git
