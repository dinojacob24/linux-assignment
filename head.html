Linux Command: head

Output the beginning of files

Home top kill ifconfig ping ssh scp curl diff head tail

Command Overview

The head command displays the beginning of a text file or piped data. By default, it shows the first 10 lines, but this can be customized.

Basic Syntax

head file              # Show first 10 lines
head -n N file         # Show first N lines
head -c N file         # Show first N bytes
head -n -N file        # Show all but last N lines

Common Options

  • -n, --lines - Output the first N lines
  • -c, --bytes - Output the first N bytes
  • -q, --quiet - Never print headers
  • -v, --verbose - Always print headers
  • -z, --zero-terminated - Line delimiter is NUL

Common Examples

# Show first 10 lines of a file
head file.txt

# Show first 5 lines
head -n 5 file.txt

# Show first 100 bytes
head -c 100 file.txt

# Show all but last 5 lines
head -n -5 file.txt

# Show headers for multiple files
head -v file1.txt file2.txt

# Show first 20 lines of multiple files
head -n 20 *.txt

Using with Pipes

# Show first 5 processes
ps aux | head -n 5

# Show first few directory entries
ls -l | head

# Show beginning of command output
cat large_file.txt | head

# Show first few matches
grep "error" log.txt | head -n 3

Pro Tips

  • Use with sort to see top entries

Basic Usage

Option Description Example
-n N Display first N lines head -n 20 file.txt
-c N Display first N bytes head -c 100 file.txt
-q Quiet mode (no headers) head -q file1.txt file2.txt
-v Verbose (always show headers) head -v file.txt

Advanced Usage

# Display first 1000 bytes
head -c 1000 file.txt

# Show first 50 lines from multiple files
head -n 50 file1.txt file2.txt

# Display first 10 lines without headers
head -q -n 10 *.txt

# Show first N-1 lines (skip last line)
head -n -1 file.txt

# Pipe with other commands
ps aux | head -n 5

# Monitor file beginning
tail -f file.txt | head -n 20

# Display first megabyte
head -c 1M large_file.dat

# Show beginning of compressed file
zcat file.gz | head -n 20

Scripting Examples

# Process first N lines of each file
process_heads() {
    for file in "$@"; do
        echo "=== Processing $file ==="
        head -n 10 "$file" | while read -r line; do
            echo "Line: $line"
            # Process each line here
        done
    done
}

# Monitor file headers
monitor_headers() {
    while true; do
        clear
        head -n 5 "$1"
        sleep 1
    done
}

# Extract file headers
extract_headers() {
    for file in "$@"; do
        head -n 1 "$file" > "${file}.header"
    done
}

# Compare file beginnings
compare_heads() {
    if diff <(head -n 10 "$1") <(head -n 10 "$2") >/dev/null; then
        echo "First 10 lines match"
    else
        echo "Files differ in first 10 lines"
    fi
}

Common Tasks

# Preview large files
head -n 100 large_file.txt

# Check file headers
head -n 1 *.csv

# Quick file inspection
head -v -n 5 /var/log/*.log

# Extract sample data
head -n 1000 data.csv > sample.csv

# View file start in hex
head -c 100 binary_file | xxd

# Check multiple log files
head -n 20 /var/log/{syslog,auth.log}

# Monitor log beginning
watch -n 1 'head -n 20 /var/log/syslog'

# Extract file metadata
head -c 512 image.jpg > header.bin

Performance Considerations

  • Memory Usage:
    • Uses minimal memory regardless of file size
    • Efficient for large file previews
    • Stops reading after required lines
  • Processing Speed:
    • Very fast operation
    • No file seeking required
    • Efficient with compressed files
  • Best Practices:
    • Use -c for binary files
    • Prefer -q for multiple files
    • Consider compression

Troubleshooting

  • Common Issues:
    • File permission denied
    • Invalid byte count
    • Binary file output
    • Character encoding issues
  • Solutions:
    • Check file permissions
    • Use -c for binary files
    • Specify correct encoding
    • Use -q for clean output
